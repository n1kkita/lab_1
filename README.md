# Лабораторна робота 1.3 — Блочна верстка

**Тема сайту:** кав'ярня-читальня «Кава та Книга».

## Виконані завдання

### 1.3.1 Позиціонування та макети
- **Блочні елементи:** використано семантичні блоки `header`, `nav`, `main`, `aside`, `section`, `footer` та контейнери `div`.
- **Макет фіксованої ширини:** контейнер `#page-wrapper.layout-fixed` має ширину **960px**, вирівняний по центру (`margin: 0 auto`).
- **Макет гнучкий (розтягуваний):** секція `#promo-section.layout-fluid` займає 100% ширини батька, внутрішній контент обмежений `max-width: 960px`.
- **1 колонка:** шапка, навігація, підвал.
- **2 колонки:** блок «Акції тижня» (`.two-columns`).
- **3 колонки:** основний контент (лівий сайдбар 22%, центр 56%, правий сайдбар 22%).

### 1.3.2 Розмітка блоками
- Сторінка розмічена за допомогою блоків без використання таблиць: окремі блоки для шапки, навігації, основного контенту з трьома колонками, промо-секції з двома колонками та підвалу.
- (Завдання передбачає PSD-макет; у проєкті реалізовано типовий макет кав'ярні, який можна зіставити з будь-яким безкоштовним PSD-шаблоном і повторити структуру блоків.)

### 1.3.3 Перевірка в браузерах та інспектор
- Відкрийте `index.html` у **Chrome**, **Firefox**, **Edge**.
- Якщо щось з’їжджає або ламається:
  - Натисніть **F12** (або ПКМ → «Переглянути код») для відкриття інструментів розробника.
  - Вкладка **Elements** (Chrome/Edge) або **Inspector** (Firefox): переглядайте дерево HTML і стилі для кожного блоку.
  - Перевіряйте обчислені значення (Computed), відступи (margin/padding) та ширини блоків.
- Для налагодження макету використовуйте підсвічування елементів при наведенні в інспекторі.

### 1.3.4 Валідація макету
- **HTML:** [Validator W3C — Markup](https://validator.w3.org/#validate_by_upload). Завантажте файл `index.html` або вставте код — перевірте відсутність помилок.
- **CSS:** [Validator W3C — CSS](https://jigsaw.w3.org/css-validator/#validate_by_upload). Завантажте `styles.css` — переконайтеся, що помилок немає.
- У проєкті використано коректний DOCTYPE, кодування UTF-8 та семантичні теги HTML5, щоб макет проходив валідацію.

## Структура файлів
- `index.html` — головна сторінка.
- `styles.css` — стилі (фіксований і гнучкий макет, 1/2/3 колонки).
- `README.md` — ця інструкція.
